iPad / GitHub Pages Fix

Your website can work in Safari while the installed iPad app shows 404 because the home-screen icon is still pointing to an old URL or cached service worker.

1. Delete old iPad home-screen apprequired
2. Open Safari to the working root siteroot URL
3. Share → Add to Home Screen againrefresh start_url
4. If still failing, clear Safari website data for GitHub Pagescache reset
5. Upload only root files, never the zip/folderdeploy safe

This build sets manifest start_url and scope to ./ so the installed app opens the same root path as Safari.
